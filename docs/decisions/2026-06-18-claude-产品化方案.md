# AgentMesh Runtime — 分仓 / CLI 化 / Server 化 产品方案

## Context(为什么做这件事)

`/Users/ferdinandji/agent-reinforcement-system` 是从 OpenClaw 平时运行中抽象出的 agent 运行时基础设施,含 4 大能力:First-Principles 推理强化、HA 情景记忆(Neo4j→SQLite FTS→raw grep + Gemini 向量)、Autonomous-Loop/OODA、一致性与恢复。它已有成熟 CLI(`xng`)、6 篇模块文档、4 个 skill、GitHub remote,且做过一轮商业化脱敏。

目标:按 AgentMesh360 旗下产品(Lecturecast / JobAgent)的范式,把它**分仓、CLI 化、server 化**,作为生态里**第三个并列的垂直产品(开发者向 infra)**对外商业化。

已确认的产品决策:
- 定位 = **独立 infra 产品**,与 Lecturecast / JobAgent 并列。
- 与内部自用的 AgentMemory **不相关、不合并**(各走各的)。
- 开源策略 = **完整本地能力**:开源 client 的 BYOK 模式给全部 recall/向量能力,核心排序算法也开源。
- server 卖点 = **托管重依赖**(省去用户自建 Neo4j docker + 配 embedding key),按 credit 计费。
- 因"完整本地能力"决策,塌成**两仓**(无需单独私有 dev source 仓)。

预期结果:一个公开的 `AgentMesh-Runtime`(完整开源 CLI,引流)+ 一个私有的 `agentmesh-runtime-server`(托管收费),共享同一份开源核心包,复用 `agentmesh-core` 的身份/credits/订阅。

## 目标形态:两仓 + 一个共享包

| 产物 | 可见性 | License | 对标 | 职责 |
|---|---|---|---|---|
| `AgentMesh-Runtime` | PUBLIC | Apache 2.0 | `AgentMesh-Lecturecast` | 完整 CLI + 全部本地能力,BYOK/Hosted 双模;**内含**开源核心包 `agentmesh_runtime` |
| `agentmesh-runtime-server` | PRIVATE | proprietary | `lecturecast-server` | 托管 Neo4j+Gemini,credit 计量 API,多租户,接 core 计费;**依赖**(不复制)上面的核心包 |
| `agentmesh-core`(复用) | PRIVATE | — | — | 身份/credits/订阅,ECOSYSTEM.md 注册新产品 |

核心包 `agentmesh_runtime`(import 名)/ `agentmesh-runtime-core`(PyPI 名):物理 host 在 PUBLIC client 仓的 `src/agentmesh_runtime/`,server 通过 pyproject 依赖它,**零代码复制、不分叉**。

## Phase 1 — 抽公共包 `agentmesh_runtime`(在现仓内重构)

依赖图已确认强内聚核心团:`unified_memory_recall→vector_store`、`episode_ingest→sync_state`、`sync_backfill→episode_ingest`、`autonomous_loop→{recall,ingest,checkpoint}`。这一团整体进包。

文件映射(现 `src/*` → `agentmesh_runtime/*`):recall.py、vector_store.py、ingest.py、sync_state.py、sync_backfill.py、checkpoint.py、neo4j_recall.py、loop.py、rehydrate.py、kb_to_graph.py、journal_to_episode.py;`schemas/*.json` 作为 package data。`src/xng.py`、`scripts/`、`skills/`、`docs/`、`examples/` 属 client 专属,不进包。

关键重构 —— **多租户可注入 config**(让同一份代码 BYOK/server 两用):
- 在 `recall.py` 引入 `RecallConfig` dataclass(`neo4j_uri/user/password/database`, `sqlite_db`, `workspace`, `session_dirs`, `backend_weights`, `tenant_id`)。
- `recall(query, top_k, ..., config: RecallConfig | None = None)`:`config=None` 时 `from_env()`(保持 BYOK 现状 100% 兼容);server 每请求注入租户专属 config。
- `recall_neo4j` 等内部函数改为接收 driver/config,不再读 module 全局。
- 排序权重(`BACKEND_WEIGHTS`/`ENTITY_TYPE_WEIGHT`)保持模块常量(两边一致才公平)。

验证:`pip install -e .` 后 `python -c "from agentmesh_runtime.recall import recall; recall('test')"` 跑通;改造前后同 query 输出一致(diff 回归)。

## Phase 2 — 开源 client 仓 `AgentMesh-Runtime`(先开源,引流)

骨架对齐 `/Users/ferdinandji/AgentMesh-Lecturecast`:`src/agentmesh_runtime/`(核心包)+ `src/agentmesh_runtime_cli/`(CLI)+ `scripts/` + `skills/` + `docs/` + `site/`(落地页,CNAME=`runtime.agentmesh360.com`)+ `examples/` + `install.sh` + Apache `LICENSE` + `AGENTS.md` + `README.zh.md`。

**`xng.py` → 双后端改造**(`cli.py` + 新增 `backend.py`):
- 当前 `run_py()` 用 subprocess 调本地脚本 → 改为 `Backend` 抽象:
  - `LocalBackend`(BYOK,默认):直接 `import agentmesh_runtime`,无进程开销。
  - `HostedBackend`:`httpx` 调 server,`Authorization: Bearer {api_key}`,402→引导升级。
- 切换优先级:env(`AGENTMESH_RUNTIME_MODE/API_KEY/API_BASE`)> config(`~/.agentmesh/runtime/config.toml`)> 默认 BYOK。
- 新增子命令 `login` / `logout` / `mode`;`doctor` 扩展报告当前 mode 与连通性。
- `recall/ingest/embed` 可切 Hosted;`loop/rehydrate/sync/bootstrap` **始终本地**(用户机器上跑)。

CLI 改名:`xng`(=小南瓜,内部梗)对外换 **`agentmesh-runtime`**(短别名 `amr`),`[project.scripts]` 双注册;`xng` 保留为隐藏 alias 一个版本周期,README 不宣传。License:MIT→Apache 2.0(单一版权持有人,无障碍)。

验证(纯 BYOK,无 server):`install.sh` → `agentmesh-runtime doctor`(显示 byok)→ `memory recall` → `loop run examples/goal_frame.example.json` → `demo` 全绿;`docker-compose.neo4j.yml` 起本地 Neo4j 验证;GitHub Pages 落地页可访问。

## Phase 3 — 私有 server 仓 `agentmesh-runtime-server`(后补)

骨架对齐 `/Users/ferdinandji/lecturecast-server/app/`:`main.py` / `config.py` / `core_client.py`(照抄 deduct/refund/resolve)/ `db.py` / `tenancy.py` / `routes/{health,recall,ingest,embed}.py` / `services/*`。依赖 Phase 1 共享包。

路由与计费(动作码在 `agentmesh-core/app/products.py` 注册):

| 路由 | action_code | 建议成本 |
|---|---|---|
| `POST /v1/recall` | `runtime.recall` | 1(含向量 2) |
| `POST /v1/ingest` | `runtime.ingest` | 2/episode |
| `POST /v1/embed` | `runtime.embed` | 1/1k tok |
| `GET /v1/health` `/v1/whoami` | — | 免费 |

每计费路由用 core 范式:`resolve(api_key)` → `deduct(idempotency_key)` 预扣 → 干活 → 失败 `refund`,余额不足 402。鉴权优先用 core 的 `/v1/internal/credentials/resolve`(比解 JWT 正规)。

多租户隔离:**MVP 用方案 B**(Neo4j community + 节点 `tenant_id` 标签,Cypher 注入 `WHERE n.tenant_id=$tid`,通过 RecallConfig 传)+ SQLite 每租户独立文件 `/data/tenants/{account_id}/`。生产化后可迁方案 A(Neo4j Enterprise multi-database)。托管 Gemini key 由 server 持有(config.py),实现"省去配 key"卖点。

验证:本地起 server + core(dev/mock)。`curl -H "Authorization: Bearer …" POST /v1/recall` 返回结果且 core ledger 扣 1;余额 0 返 402;故障 refund 回滚;两租户互不可见。

## Phase 4 — 打通 Hosted mode 端到端

实现 `HostedBackend` + `login/logout/mode`。验证全链路:`login` → `mode` 显示 hosted → `memory recall` 走 HTTP 命中 server、core ledger 出现扣费 → `logout` 回落 BYOK 仍可本地。同 query 在 BYOK/Hosted 下排序一致。

## 开放问题(已给推荐默认,执行中可调,无需阻塞)

1. **Neo4j 隔离 A/B**:推荐 MVP 用 B(community 标签隔离,免 license 成本),生产迁 A。
2. **Hosted 付费转化**:核心 IP 已开源、BYOK 给全部能力,Hosted 卖点纯为"省运维"。护城河从"算法保密"转为"托管+数据+生态集成"——需接受这一点。
3. **共享包发布**:推荐 PyPI 公开(反正开源),server 构建最简。
4. **CLI 框架**:先保 argparse 跑通 Phase 2,typer(对齐 lecturecast)作为可选优化后置。

## 端到端验证总览

每 Phase 设 gate,上一阶段 e2e 全绿才进下一阶段。最终验收:① BYOK 全程离线可跑(doctor/recall/loop/demo);② Hosted 链路 recall 命中 server 且 core 正确扣费/退款/402;③ 同 query 两模式排序一致;④ 多租户隔离验证;⑤ 落地页 + 开源仓 grep 无私人信息。
