给你的 AI agent 加上 记忆、循环、和 恢复。

AgentMesh Runtime 是一层 agent 用的本地运行时脚手架：跨会话的持久情景记忆、有界 OODA 循环记录、崩溃后干净拾起。智能仍然来自你的 agent（Claude Code / OpenClaw / Codex / Hermes / 自定义）——这一层只做底下的脏活。

What it gives you

5 件 诚实 的事。

不是 8 件，不是 12 件。下面这些就是代码里真的实现了、能跑的。其余的请看下一段「不做什么」。

memory

持久情景记忆

Neo4j 图 → SQLite FTS5（BM25）→ 文件 grep → 可选 Gemini 向量。任一后端挂掉，其余继续服务。

loop

OODA 循环脚手架

Observe / Orient / Decide / Act / Verify / Record 状态机。每一步落到持久轨迹。决策由你的 agent 给。

recovery

崩溃 / 重启恢复

SQLite ↔ Neo4j 延迟同步账本、checkpoint 存储、rehydrate 一键重建"我刚才到哪了"。

health

一键自检

doctor 输出 JSON：Neo4j 连通、SQLite 健康、账本漂移、checkpoint 状态。

anti-pattern

规则式反模式检测

blind_retry / phantom_progress / pivot_exhaustion，是计数器和阈值，不是魔法。

+

给 agent 而非给人

整个仓库设计为 agent 调用。看 AGENTS.md，里面写了什么时候 ingest、什么时候 rehydrate、什么话不能对人类许诺。

What it is NOT

4 件 明确 不做的事。

这层的边界讲清楚，才不会让你和你的 agent 误以为它能干别的事。

not an LLM

不带 AI 模型

它给你已经在用的 agent 做脚手架。没有内置 LLM，也不打算有。

not a SaaS

不在云上跑

没有云服务、没有账户、没有 API key。完全本地。Gemini 向量召回是可选的，且 key 由你提供。

not a brain

不替你做决策

OODA 循环的 decide() 和 verify() 是有意做小的存根。真正的推理来自你的 agent。

not at scale yet

不是大规模系统

向量层是 O(n) 暴力余弦。个人 / 小团队规模够用。百万级向量请外置 pgvector / faiss。

Install & use

两条命令，开跑。

# clone + install
git clone https://github.com/jiyangnan/AgentMesh-Runtime.git
cd AgentMesh-Runtime
uv sync

# 自检
uv run agentmesh-runtime doctor

# 召回记忆 / 摄入会话
uv run agentmesh-runtime memory recall "your query" --top-k 5
uv run agentmesh-runtime memory ingest-file /path/to/session.jsonl discord

# OODA demo · 崩溃恢复
uv run agentmesh-runtime demo
uv run agentmesh-runtime rehydrate --write-default --print-path

# 短别名：amr 可替换 agentmesh-runtime

需要图后端？仓库里带一个 docker/docker-compose.neo4j.yml，docker compose up 就行。

FAQ

问到底。

它自己带 LLM 吗？
不带。这是脚手架，智能由你已经在用的 agent（Claude Code / OpenClaw / Codex / Hermes / 自定义）提供。脚手架做的是记忆、循环记录、崩溃恢复这些底层事情。
它会上云吗？需要账户或 API key 吗？
不会，也不需要。所有数据都在你自己的机器上，没有 AgentMesh360 账户、云服务或 key 要求。可选的 Gemini 向量召回需要你自己的 GEMINI_API_KEY。
OODA 循环会替我决策吗？
不会。decide 和 verify 步骤是有意做小的状态机存根——它们记录你做了什么，但不替你想下一步。真正的推理由你的 agent 出。
我用的不是 OpenClaw，也能用吗？
可以。默认路径还指向 ~/.openclaw/* 是历史原因，把 ARS_SESSION_BASE 和 ARS_MEMORY_DB 改成你自己 agent 的路径即可。Claude Code / Codex / Hermes / 任何自定义 agent 都能驱动。
未来会做托管服务吗？
暂时不做。如果有真实用户问"能帮我托管 Neo4j 吗"，会做一个**只托管数据库**的极薄 server——不是智能层。决策见 docs/decisions/。
